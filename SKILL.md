# 🎬 AI视频 分镜提示词生成器（通用方法论）

帮助用户将想法转化为专业的 AI 视频分镜提示词。**适用于 Kling（可灵）、Seedance（即梦）、Sora、Runway 等主流 AI 视频平台。**

> 📚 **本 skill 附带参考文件**：`references/lens-techniques-38.md`（38种运镜技巧）、`references/seedance-prompt-patterns.md`（Seedance 2.0提示词模式）、`templates/storyboard-template.md`（⚠️旧版五层框架，仅参考7种场景类型，输出格式以本文档 Step 2 七要素为准）、`references/kling-storyboard-workflow.md`（Kling实战工作流）。

> ⚠️ **平台切换须知**：不同平台语法不同（Kling用自然中文，Seedance需`@素材`语法，写实人脸限制仅限Seedance）。**调用前确认用户目标平台**。

## Seedance 2.0 核心能力

### 技术参数

| 输入类型 | 格式要求                        | 数量限制             | 大小限制 |
| -------- | ------------------------------- | -------------------- | -------- |
| **图片** | jpeg、png、webp、bmp、tiff、gif | ≤ 9 张               | < 30 MB  |
| **视频** | mp4、mov                        | ≤ 3 个，总时长 2-15s | < 50 MB  |
| **音频** | mp3、wav                        | ≤ 3 个，总时长 ≤ 15s | < 15 MB  |
| **文本** | 自然语言                        | -                    | -        |

**混合输入总上限：12 个文件**

> 💡 建议：优先上传对画面或节奏影响最大的素材。有参考视频会消耗更多生成额度。

**核心功能**：

1. **参考图像** - 精准还原画面构图、角色细节、服装样式
2. **参考视频** - 支持镜头语言、复杂动作节奏、创意特效复刻
3. **视频延长** - 平滑延长与衔接，可"接着拍"
4. **视频编辑** - 角色更替、剧情颠覆、片段调整、场景融合
5. **多模态组合** - 图+视频+音频+文本自由组合创作

**基础能力增强**：

- ✅ 物理规律更合理、动作表现更自然流畅
- ✅ 指令理解更精准、风格保持更稳定
- ✅ 支持复杂动作、连续运动等高难度生成任务

**交互方式**：使用"@素材名"指定用途，如"@图片1 作为首帧，参考@视频1 的运镜"

**入口说明**：

- **首尾帧入口**：只上传首帧图 + prompt
- **全能参考入口**：多模态（图、视频、音频、文本）组合输入

## 分步引导流程

### Step 0：确认目标平台（先于任何内容生成）

在开始写分镜前，**必须先确认用户的目标平台**，因为不同平台的输出格式、限制、语法完全不同。直接问用户：「你想用哪个 AI 视频工具？可灵/Kling、即梦/Seedance、Sora，还是其他？」

如果用户不确定，推荐 Kling（可灵）作为默认——兼容性最好，中文提示词支持完善。

> 📍 **检查点**：确认目标平台后，引导用户回答以下问题 → 收集完毕后向用户展示摘要确认 → 确认后进入 Step 1。

---

### Step 1：理解用户想法 + 深入挖掘细节

#### 快速收集

用友好的方式询问用户：
1. **你想讲什么故事？**（一句话概括核心内容）

> ⚠️ **摄像机/手机位置歧义陷阱**：当用户说"手机放地上""手机放在桌上"等描述时，**必须追问镜头朝向**——是镜头朝上仰拍（从地面低角度拍到人物），还是镜头朝下俯拍（拍到手机本身）。这两种完全不同的视角会产生截然不同的画面效果。例如"手机放地上"可以是：
> - 镜头朝上 → 低角度仰拍人物腿部/身体（常用于避免走光、展示大长腿）
> - 镜头朝下 → 俯拍手机屏幕/周围环境
> **绝不能自行假设镜头朝向**，必须在 Step 1 就确认清楚。
>
> 🎯 **抖音/TikTok 舞蹈视频的常见开场模式**：手机放在地上镜头朝上仰拍，先低角度拍到舞者坐姿/蹲姿的大长腿（避免走光），然后捡起手机转为自拍视角开始跳舞。这个"地面仰拍→捡手机→自拍跳舞"的视角转换是抖音舞蹈视频的经典叙事创意，遇到类似需求时应主动向用户确认是否是这个模式。
2. **视频时长？**（4-15秒，默认15秒）
3. **参考素材情况？**（有无可用的图片/视频/音频参考）

#### 深入挖掘维度

根据用户的初步回答，逐步引导完善以下维度：

#### 1. 内容叙事

- 故事的起承转合是什么？
- 关键情节有哪些？
- 需要哪些角色/主体？
- 对白/旁白内容？

#### 2. 视觉风格

- 整体风格：写实/动画/水墨/科幻/复古/电影感？
- 色调氛围：明亮/昏暗/暖色/冷色/黑白？
- 画面比例：竖屏(9:16)/横屏(16:9)/电影宽屏(2.35:1)？

> 💡 **叙事色彩弧（Narrative Color Arc）**：当故事跨越多段情绪段落时，可以用色调变化来驱动情感叙事。经典三段式结构：现实压抑段（冷色调）→ 回忆幸福段（暖色调）→ 回归现实/崩坏（冷色调渐黑）。在七要素的「色彩与影调」字段中显式声明色温转换规则；如果单个分镜内有色调转变，在该分镜的 🎥 视频动作描述的「色彩与影调」字段中描述色调变化轨迹。参见 `references/kling-storyboard-workflow.md` 的BGM对齐+色调联动方法。

#### 3. 镜头语言

- 景别变化：远景→全景→中景→近景→特写？
- 运镜方式：推/拉/摇/移/跟/环绕/升降/希区柯克变焦？
- 转场方式：硬切/渐变/匹配剪辑/特效转场？

#### 4. 动作与节奏

- 主体动作：行走/奔跑/打斗/舞蹈/表情变化？
- 动作节奏：舒缓/急促/有张有弛？
- 是否需要音乐卡点？

#### 5. 声音设计

- 配乐风格：激昂/温馨/悬疑/欢快/史诗？
- 音效需求：环境音/动作音效/特殊音效？
- 对白/旁白：有台词吗？什么语气？

> 📍 **检查点**：向用户展示收集到的完整创作意图摘要（故事梗概+风格+时长+参考素材），确认后进入 Step 2 构建分镜。

### Step 2：构建分镜结构（七要素融入每个分镜 + 电影参考锚点）

> 🎯 **本步核心**：五层框架只作为 Agent 内部"前期准备清单"（preparatory worksheet）使用，**不输出到最终分镜脚本**。最终分镜脚本唯一输出格式 = 每个分镜内部包含完整的七要素结构 + 电影参考锚点。

#### 前期准备清单（仅用于 Agent 内部理清，不输出到最终脚本）

> 📝 五层准备清单（Agent内部思考工具，不输出到最终脚本）：
> 第1层 核心风格定位 | 第2层 角色与场景锚点 | 第3层 氛围与质感 | 第4层 摄影机规则 | 第5层 分秒级时间轴
> 完整填写指南见 `templates/storyboard-template.md`

> ⚠️ **五层准备清单只用于 Agent 内部思考与风格对齐，不输出到最终分镜脚本**。最终分镜脚本只输出七要素结构（见下文「七要素结构」和 Step 3）。

#### 电影参考锚点（强制必填项）

| 触发条件 | 操作 |
|---------|------|
| ✅ 用户明确提到参考电影 | 提取该电影的风格 DNA → 填入七要素的「风格核心」+「视觉基调」+「色彩与影调」 |
| ❌ 用户未提到参考电影 | AI 必须主动推荐 1-2 部风格匹配的电影 → 填入七要素 + 在准备清单中标注「推荐参考」 |

```
- **风格核心**：电影参考《XXX》[导演YYY / 摄影ZZZ]，[流派标签]、[质感关键词]、[杜绝什么]。[核心氛围反差]
```

#### 🖼️ 场景建立帧（Scene Establishing Shot）—— 环境一致性锚点

> ⚠️ **为什么需要场景建立帧？** 多分镜视频最常见的翻车点就是「环境跳变」——分镜1是暖光木地板、分镜2变冷光水泥地、分镜3色调突变。根本原因：各分镜的环境描述各自独立，缺少统一参考锚点。**场景建立帧 = 在正式分镜之前，先定义一个纯环境的「锚点图」**，后续所有分镜的环境要素必须与此帧对齐。

**执行时机**：在分镜汇总表确认后、展开七要素分镜之前（Step 2.5 → Step 3 之间）。

**场景建立帧的内容结构**（纯环境描述，**不含任何角色**）：

```markdown
## 🖼️ Scene Establishing Shot（场景建立帧）

> 本帧用于先生成一张纯环境效果图，作为所有后续分镜的环境参考锚点。
> ⚠️ 禁止包含任何角色描述。

**英文 Prompt（用于 MJ/SD 生图）**：
[纯英文场景描述，包含以下全部要素] --ar [画幅比] --v 6.0

**环境要素清单**（Agent 必须逐项填写）：
- 🏠 **空间布局**：整体空间的形状、大小、进深、边界（如：15㎡矩形房间，3米层高，左侧落地窗）
- 💡 **光源位置与色温**：主光源位置 + 色温(K) + 补光位置（如：右侧45°落地灯3200K暖光 + 顶部冷白射灯5000K）
- 🎨 **色调**：整体色彩基调（如：暖橙+灰蓝双色调，低饱和度）
- 🧱 **地面/墙面材质**：具体材质+颜色+质感（如：浅灰哑光水泥地面 + 白色微水泥墙面）
- 🪑 **主要道具/物件位置**：关键道具的空间坐标（如：画面中央偏右放置一张胡桃木圆桌）
- 🌤️ **天气/时间**：室外天气或室内时间段暗示（如：雨天傍晚，窗外霓虹透入 / 晴天正午，强侧光）

**中文环境描述**（供 Agent 内部对齐用）：
[基于以上要素的综合场景描述，200字以内]
```

**场景建立帧 → 分镜联动规则**：

1. **每个分镜的「色彩与影调」**：色调、布光方案必须与场景建立帧保持一致，除非该分镜显式标注了色调变化（如叙事色彩弧转换）
2. **每个分镜的「画面内容」环境部分**：地面、墙面、道具描述必须与场景建立帧的布局对齐，仅允许因运镜角度不同而产生的视角差异
3. **分镜间环境连续性**：如果两个相邻分镜在同一场景内，后一个分镜的环境描述必须从前一个分镜的环境状态延续，不能凭空切换
4. **多场景故事**：如果故事跨多个场景（如室内→室外），每个场景需要独立的场景建立帧，并在分镜汇总表中标注场景切换点

#### 分镜内七要素结构（唯一输出格式）

每个分镜的**生图帧描述**和**视频动作描述**都必须以**七要素结构**呈现（完整结构模板见 Step 3）：

| 要素 | 内容 | 来源层 |
|------|------|------|
| **风格核心** | 电影锚点 + 流派标签 + 质感关键词 + 杜绝什么 + 核心氛围反差 | 电影参考锚点（来自「电影参考锚点」section） |
| **视觉基调** | 画幅 / 胶片机型 / 镜头型号 + 特殊效果 | 电影参考锚点 |
| **色彩与影调** | 时代美学 + 主色+对比色 + 颗粒/滤镜 + 布光法则 | 电影参考锚点 |
| **景别** | 机位角度 + 人物比例 + 拍摄方位 | 单镜调度 |
| **构图** | 主体位置 + 前景/背景关系 | 单镜调度 |
| **运镜手法** | 焦距 + 光圈 + 运动方式 + 速度 + 方向（必须包含 ≥3 个参数维度）| 单镜调度 |
| **画面内容** | 角色姿态/动作 + 关键道具/环境 + 光影在角色上的呈现 | 单镜内容 |

> 🔒 **铁律**：禁止输出"全局声明段落"——所有氛围、画质、风格信息必须**溶解**到每个分镜的七要素内部。每个分镜内部必须自包含、可独立提交给 AI 视频/图片模型生成。即使单独拿出一个分镜，模型也能获得完整的氛围/画质/镜头上下文。

#### 📐 镜头参数与布光参考速查（构建七要素时的辅助工具）

#### 🚨 分镜定义铁律（用户修正痛点，必须遵守）

**一个分镜 = 一个连续的摄像机镜头，只有一个机位、一个场景、一个时间段。**

| ✅ 正确 | ❌ 错误 |
|---------|---------|
| 特写安妮的眼睛，4秒 | 夜市吃东西+别花婚纱亲密瞬间，5秒 |
| 中景跟拍她在海边奔跑，4秒 | 三段回忆快切蒙太奇，6秒 |
| 近景她低头打字回复，4秒 | 从悲伤到回忆到崩坏，8秒 |

**常见违规模式**：
- ❌ 把多个场景塞进一个分镜（夜市+婚纱=两个分镜）
- ❌ 一个分镜里让角色"从A走到B然后做C"（应拆为走+做两个分镜）
- ❌ 一个分镜里做多段快切蒙太奇（应拆为独立分镜）

**每个分镜只包含**：一个机位 × 一个动作/表情 × 一个持续时间段。

如果故事需要多个场景（如三段回忆），每个回忆场景单独一个分镜，不能合并。

#### 时间轴约束（按平台）

不同平台单段时长上限不同，分镜时长必须遵守：

| 平台 | 单段最大 | 建议单段 | 提示词形态 |
|------|---------|---------|-----------|
| Kling（可灵） | 5s | 3-5s | 独立分段，每个提示词自包含 |
| Seedance（即梦） | 15s | 10-15s | 连续时间轴描述 |
| Sora | 60s | 10-20s | 英文连续描述 |

**铁律**：每个分镜的时长不可超过目标平台的单段上限。Kling 每段必须 ≤5s。

> 📍 **检查点**：先向用户展示分镜汇总表（首行为场景建立帧）+ 场景建立帧的环境要素清单，确认环境锚点和分镜节奏后，再展开每个分镜的七要素细节。

| 五层维度 | 对应七要素 | 溶解方式 |
|---------|----------|---------|

#### AI 视频提示词书写原则（跨平台通用）

1. **七要素不可省略**：每个分镜必须完整呈现七个标签字段；**禁止返回平铺式散文描述**
2. **电影锚点首字段**：「风格核心」必须以电影参考《XXX》[导演/摄影]开头
3. **角色锚点嵌入**：每个分镜的「画面内容」字段必须包含发色+发型+眼睛颜色+服装，确保角色跨片段一致
4. **服装要显式声明**：如果故事中有多套服装，每个分镜必须写明当前穿什么
5. **一个镜头说清楚**：不要写"先A后B"，每段只能做一个连续动作
6. **文字特效（如眼中LED文字）**：在提示词中写"字符逐个亮起闪烁，如LED霓虹灯管矩阵点阵屏，每个字独立点燃有发光光晕"这种具体描述，AI 视频模型能理解逐帧动画效果
7. **≤200字/段**：精炼有效，避免模糊描述
8. **运镜参数三维度最低标准**：「运镜手法」字段必须包含以下 ≥3 项：焦距（mm）、光圈（F值）、运动速度（cm/s或°/s）、运动方向/轨迹、距离（m）。禁止只写"推近""拉远""跟拍"而不给具体参数
9. **环境一致性锚定（🆕）**：所有分镜的环境描述（光源、色调、地面/墙面材质、道具位置）必须与「场景建立帧」严格对齐。「色彩与影调」和「画面内容」中的环境部分不得凭空引入场景建立帧中未声明的元素。如需环境变化（如灯灭了、天黑了），必须在该分镜的「场景锚点」字段中显式说明变化原因和变化后状态

##### 范例 B：机器人骑鸵鸟（运动叙事场景）

> 风格：原子朋克 + 运动喜剧。电影锚点：《疯狂的麦克斯4》/《僵尸肖恩》。

范例 B 关键要点：
- 运动场景下，🎥 段的「画面内容」必须描述**颠簸/抖动的物理细节**（鸵鸟奔跑产生剧烈抖动 → 机器人身体高频颠动）


**适用场景**：伤感开场、悬疑开场、角色内心世界先于外部环境被观众看到

##### 技巧2：眼中LED文字逐字闪烁（结尾特效）

当需要在角色眼睛中呈现文字时，用 LED 二极管/霓虹灯管效果代替整体闪现：

```
瞳孔中浮现LED霓虹灯管/发光二极管矩阵效果——白色发光文字像点阵屏跑马灯一样，
逐字逐个亮起闪烁：先亮起「我」→「爱」→「你」（逐个点燃，每个字有霓虹灯管
通电发光的闪烁感、光晕辉光和电子脉冲感），随即所有字同时熄灭，再逐字亮起
「再」→「见」（同样逐个闪烁点亮），发光字在虹膜上投影出霓虹光晕。
然后眼睛缓缓眨下，画面渐黑。
```

**原理**：通过描述"逐字点亮""跑马灯""点阵屏"等具体视觉概念，让 AI 视频模型理解逐帧动画意图，而非静态叠加文字。

##### 技巧3：雨滴触发回忆转场

用雨滴击中眼睛作为现实↔回忆的转场锚点：

```
极特写[眼睛颜色]眼球，雨滴慢镜头击中眼球表面瞬间水滴扩散覆盖虹膜，
闪白过渡，[切入/切出回忆]
```

##### 技巧4：CRT老电视雪花噪点转场（回忆蒙太奇 / 80-90年代复古过渡）

当需要在多段回忆之间做80-90年代复古转场，或从回忆回到现实时，用CRT静电雪花噪点代替闪白：

```
画面逐渐出现大量黑白雪花噪点和静电条纹横纹，如同老式CRT显像管电视信号丢失的雪花屏效果。雪花噪点持续0.5-1秒后逐渐消退，画面从雪花中重新清晰浮现，过渡到下一个场景。过渡期间BGM保持连续（雪花画面处可短暂静默或只有微弱静电杂音），雪花消退后画面恢复正常色调。
```

**适用场景**：多段回忆蒙太奇切换、80-90年代怀旧氛围、回忆↔现实的转场锚点

**vs. 闪白**：

| 转场 | 速度 | 情绪 | 适用 |
|------|------|------|------|
| 闪白 | 0.1-0.3s，快速 | 冲击/觉醒 | 现实↔回忆单次切换 |
| CRT雪花 | 0.5-1s，慢速 | 怀旧/失真 | 多段回忆之间的过渡 |

## 🌀 迭代式分镜修订工作流 (Iterative Storyboard Revision)

当对已有分镜脚本进行版本升级（如 v1→v2→v3）时，**不能从头重写**——需要保留已验证的内容，精准修改目标部分。以下是经过实战验证的修订工作流：

### 修订类型矩阵

| 修订类型 | 示例 | 操作方式 |
|---------|------|---------|
| **全局重构** | 加5层框架、改平台格式、加运镜编号 | 新生成第1-4层 + 逐分镜改写第5层 |
| **结构插入** | 在回忆之间插入CRT转场 | 找到插入点，逐分镜后移，重算BGM时间戳 |
| **局部增强** | 给每个分镜加🖼️背景prompt | 遍历每个分镜，在🎞️前插入🖼️段 |
| **冻结保留** | 分镜11-12保持原始内容完全不变 | 标记为`⚠️ 保留原始版本，完全不变`，不修改 |
| **内容追加** | 在原有结局后追加2个新分镜作为真结局 | 在分镜表最末追加，更新总时长和汇总表 |
| **运镜编号** | 为每个分镜标注"运镜参考：#X" | 在分镜标题行添加运镜#标注 |
| **结构化重构** | 把旧版平铺式分镜改写为七要素结构化格式 | 遍历每个分镜，将氛围/画质/镜头信息重组为七要素标签格式；为每个分镜补充电影参考锚点；将"全局声明段落"溶解到每个分镜内部 |

### 🚨 BGM时间戳重算铁律（最容易出错的一步）

**当在已有分镜之间插入转场或分镜时，该位置之后的所有BGM时间戳都必须重算。**

重算公式：
```
新时间戳 = 原时间戳 + 插入转场/分镜的累计时长总和
```

**实战案例**：v1→v3 在06-07间插3段CRT(各0.8s)，末尾追加分镜13(4s)+14(5s)。每插一段，后续所有分镜BGM +0.8s累加。
```
受影响分镜: 07(+0.8s)→08(+1.6s)→09(+2.4s)→12(+2.4s) | 13新增(4s) | 14新增(5s)
验证: 原结束46s + 插入2.4s + 追加9s = 57.4≈58s ✓
```

### 🧊 冻结保留（Freeze Shots）

当用户要求"保留原有XX个分镜完全不变"时：从原始文件逐字复制该分镜内容，添加标记 `> ⚠️ **以下内容保留原始vX版本，完全不变**`。仅允许改BGM时间戳和运镜#标注，不允许改🎞️/🎥内容或时长。验证：`diff` 对比确认一致。

### 🎬 追加结局模式（Appended Ending）

1. 原结局的"结束感"必须保留
2. 追加分镜起始状态从"已完成"开始（渐黑→黑屏渐亮，闭眼→睁眼/新视角）
3. 追加分镜形成独立情绪弧，如：闭眼→渐黑（沉重）→ 黑屏→车内渐亮→轻笑→圆形渐黑（荒诞余韵）
4. 汇总表必须更新，追加分镜用 `★新增` 标记

### 📋 版本升级清单模板

从vN升级到vN+1时，先列出变更清单（示例）：

```markdown
# 版本变更清单：v1 → v3
## 全局变更：添加5层框架 / 添加🖼️背景prompt / 添加运镜# / 插入CRT转场 / 冻结保留分镜XX-XX
## 分镜级变更：分镜01-10增强写作质量 / 分镜06-08听觉反差 / 分镜XX更新BGM
## 追加内容：分镜13-14新增
## 汇总表：重算BGM时间戳 + 更新汇总表 + 更新总时长
```

### ⏱ 长分镜重写的拆分委派铁律（>8 分镜或 >500 行）

当对已有分镜脚本进行大规模结构化重写（如 v3→v4 平铺式→七要素结构化）时，单次 `delegate_task` 的 600s 超时**不足以完成**。**必须按以下规则拆分委派**：

| 触发条件 | 拆分方式 |
|---------|---------|
| 分镜数 ≤ 8 或目标文件 ≤ 500 行 | 单次委派可完成 |
| 分镜数 9-14 或目标文件 500-800 行 | **拆为 Part 1（前半）+ Part 2（后半）** |
| 分镜数 > 14 或目标文件 > 800 行 | 拆为 3+ 部分 |

**拆分执行步骤**：

1. **Part 1 委派**：让子 Agent 写入新文件（如 v4 路径），完成前半分镜 + 准备清单 + 转场段
2. **Part 2 委派**：让子 Agent 用 `patch` 或 `read_file` + `write_file` 在文件末尾**追加**后半分镜 + 更新汇总表
3. **端到端验证（Main Agent 必做）**：Part 2 完成后用 `read_file` / `search_files` 读取完整文件，核对分镜总数、七要素覆盖率（≥ 分镜数 × 7）、电影锚点出现次数、BGM 时间戳连续性、原文件未被修改（`mtime` 对比）。

> **反面案例**：不拆分 → 单次 600s 超时 → 触发委派超时规则 → 只能延长等待/异步回收/重置重发。

## 🔄 用户修正响应模式（子Agent交付后）

当用户对子Agent交付的分镜提出修正意见时，Main Agent 必须作为"质检"角色完整走以下闭环：

### 修正接收 → 分析 → 再委托 → 验证

1. **接收修正**：用户说"少了X""Y不对"时，先承认问题，不要辩解
2. **分析范围**：是局部修正还是全局修正？
   - 局部修正（只改某几个分镜）：用 `patch` 精准修改
   - 全局修正（如加镜头参数/加灯光/改结构）：重新委托 `@videographer`
3. **再委托时**：保留之前已做对的内容不动，只新增修正点
4. **验证**：修正后必须重新读取完整文件确认全部修改生效

### 本 session 中出现的用户修正类型（预计算法）

| 修正类别 | 触发信号 | 修正方向 | 响应方式 |
|---------|---------|---------|---------|
| 分镜结构错误 | "你知道啥叫分镜吗" | 多场景塞进一个分镜→拆为独立分镜 | 重新拆解分镜结构 |
| 元素缺失 | "灯光也没有，服了" | 漏了镜头参数/布光/微动作 | 补全缺失维度，每个分镜检查 |
| 序列错误 | "第一个镜头是从眼睛特写开始" | 开场镜头的选择错了 | 替换指定分镜的内容 |
| 效果不符 | "文字是一个个闪过" | 文字呈现方式不对 | 修改分镜描述，加入具体效果参考 |
| 内容越权 | "为什么没分配合适的agent" | Main Agent 自己写了本应委托的内容 | 停止自写，重新走委托流程 |
| **语义误解** | "谁和你说我要删除旧分镜了" | "增加"被理解为"替换/删除" | 用"增加""追加""添加"时不删现有内容，有歧义时 `clarify` 确认 |
| **摄像机方向误解** | "手机放地上意思是镜头朝上拍腿" | "手机/摄像机放地上"被理解为俯拍手机屏幕，实际是镜头朝上仰拍 | 涉及手机/摄像机位置描述时，必须追问镜头朝向（朝上仰拍 vs 朝下俯拍），不可自行假设 |
| **环境一致性缺失** | "没有环境一致性的能力，应该有一个生图帧先生成整体场景" | 各分镜环境描述各自独立，缺少统一参考锚点导致场景跳变 | 已通过「场景建立帧」机制解决：分镜汇总表首行输出纯环境锚点图，后续所有分镜的环境要素必须与此帧对齐 |

### 修正铁律

- **用户的愤怒是最优先级信号**：每次用户说"服了""你知道啥叫X吗"这类表达时，意味着当前交付物有根本性问题，需要重新检查而不是微调
- **每次修正只做一件事**：不要在一个修正循环里同时修结构+加参数+改效果，逐个确认
- **修正后必须完整验证**：不要假设只改了目标位置就完了

当你收到"写分镜"或"做视频提示词"类任务时，**你不得自行编写分镜内容**。必须执行以下三步：

1. **确认目标平台**：先问用户用哪个 AI 视频工具（Kling/Seedance/Sora等）
2. **角色分析前置**（可选）：如果用户提供了角色参考图但当前模型不能看图，先用 `mmx vision describe --image <path> --prompt "详细描述角色全部外观特征" --output json` 提取角色外观描述
3. **完整委托**：将以下信息完整打包传给 `@videographer`：
   - 故事稿/剧本
   - 角色外观描述（从 mmx 提取，含多套服装切换说明）
   - 目标平台 + 该平台的单段时长上限
   - 角色参考图路径（让子Agent也能用mmx查看）
   - BGM信息和对齐要求
   - **总时长约束**（如需≤60s，明确告知）
   - **分镜定义约束**：每个分镜=一个连续镜头，禁止合并多场景

> 即使你觉得"分镜我也能写"，也必须走委托流程。本 skill 设计为被 `@videographer` 调用，非由 Main Agent 直接使用。

#### 🔒 输出格式铁律（v3.0 新增）

> 遵循以上铁律（输出格式：七要素结构化，禁止平铺散文；电影参考：每分镜必含锚点；全局声明：氛围/画质/风格信息溶解到每分镜七要素内部）。

## ⚡ 角色分析 → 提示词嵌入工作流（跨平台通用）

> 当用户提供角色参考图时，此工作流确保多段视频的角色一致性。从外部来源提取参考知识时：广度优先于精选（完整数据 > Featured），假设需用更多数据验证。

### 步骤 1：用 mmx vision 提取角色外观（主模型不能看图时使用）

```bash
mmx vision describe --image "/path/to/ref.png" --prompt "详细描述这个角色的全部外观特征：发型、发色、服装、妆容、眼睛颜色、脸型、气质、整体风格" --output json
```

### 步骤 2：提炼"角色锚点"

从 mmx 输出中提取 5 个核心维度作为锚点：
- **发色/发型**：精确到颜色+长度+造型
- **眼睛**：精确到颜色+质感+神态
- **脸型/皮肤**：脸型+肤色
- **妆容**：关键特征（如哭泣妆、水光唇）
- **服装标识**：日常装 + 特殊场合装

### 步骤 3：锚点嵌入每个分镜提示词

每个分镜的提示词**首句**必须包含角色锚点，格式化为：
```
[发色发型+服装标识]角色[动作/表情]，[环境]，[镜头运动]，[光影色调]
```

### 步骤 4：服装切换管理

如果故事中有多套服装（如现实线vs回忆线），在每个分镜提示词中**显式标明当前服装**，避免可灵混淆。
